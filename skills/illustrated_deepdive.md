# 图文精读稿生成规范（Illustrated Deep-Dive）

当执行模式 A（默认）时，把一期视频做成「可直接发公众号的图文精读稿」，严格遵守本规范。
本规范是工作流的主力产物标准；另一种产物是对话体逐字稿（见 `dialogue_transcript.md`）。

## 核心定位

图文精读稿 = 观点提炼（深度重构，不逐句还原）+ 完整图文（叙事章节 + 密集媒体占位）+ 公众号发布全套要素（标题 / 摘要 / 封面 Prompt）。
产出目标：一篇打开即可排版发布、且每一句都经过原始字幕核校的高密度图文长文。

**共同底线**：所有事实、数字、专有名词、因果逻辑必须能在 `transcript.json` 中找到出处。定稿前必须执行 CLAUDE.md 的 **Step 2.5 对照字幕核校**。

---

## 输出结构（严格按此顺序，共 9 段）

1. 标题（含系列标记）
2. 摘要钩子
3. 封面图 Recraft Prompt
4. 内容与价值概括 ⭐**双锚点之一·必须有**
5. 嘉宾背景
6. 核心观点
7. 重构正文（叙事章节，含减负装置）
8. 结尾收束段 ⭐**双锚点之二·必须有**
9. （如有）下集预告 / 系列回链

> ⭐ **双锚点（强制）**：第 4「内容与价值概括」与第 8「结尾收束段」是一前一后两个锚点，**两者都必须有，不可省略**。它们替代"把核心观点砸在开头"的做法——开头用价值概括给"看什么 / 为何读"（轻量预告、不剧透），结尾用收束段给"判断 + 开放结局"（读完之后才下结论）。这是长科普文解决"读完没收获感"的关键。

---

### 1. 标题

- 一次生成 **5 个候选**，覆盖不同钩子类型，供用户挑选：
  1. 冲击 / 反共识型（名人 + 金额 + 争议，打开率最高）
  2. 金句型（点出核心主张）
  3. 悬念 / 对立型（"所有人 vs 他"）
  4. 设问 / 科普型（门槛低、稳）
  5. 干货 / 深度型（体现深度定位）
- 系列内容必须在标题末尾标注 `（上集）`/`（下集）` 或系列名。
- 给出推荐项 + 一句话理由（哪种调性选哪个）。

### 2. 摘要钩子

- 定位：公众号标题下方的摘要，**唯一 KPI 是"勾人点进去"，不是概括内容**。
- 要求：≤20 字优先；带悬念 / 冲突 / 利益相关，避免"目录式"罗列专业名词。
- 优先把读者拉进来（如"你正在用的 X"），从旁观者变利益相关者。
- 与标题分工互补、不重复：标题给"是谁 + 什么观点"，摘要"加注下重码"。

### 3. 封面图 Recraft Prompt

- 固定参数行：`模型 recraftv3｜风格 Motion Blur｜尺寸 2048×1024`。
- 输出可直接复制的**英文 prompt**，务必含 `no text`（中文标题后期叠加，避免乱码）；纯抽象版另加 `no people`。
- **系列统一视觉**：同一系列共享构图母题 + 配色 + 风格词，只改一个差异点区分集数。
  （示例母题：左乱 → 右清晰的横向过渡；deep navy/black 背景 + electric blue 主光 + amber 点缀。）
- 主选 + 备选各一版（概念抽象版 / 人物剪影版），并提示可调项（冷暖、抽象度）。

### 4. 内容与价值概括（双锚点之一）

- **只讲"事"**：这篇文章讲什么（用 2-3 个关键钩子串起来，不流水账）+ 为什么值得读 + 嘉宾核心主张。
- **严禁与"嘉宾背景"重复**：人物履历（头衔、过往成就、近况）一律交给第 5 部分，本部分不出现。
- 用加粗压住"核心主张"那一句。
- 4 段以内，每段可独立成立。
- **结尾附「路线条」导航（长文必备）**：本段末尾、进正文之前，放一个 `📍 本文路线：A → B → C → D → E` 的 callout（浅灰底、📍 图标，**不是标题**），用箭头串起各大章节关键词，给读者一张"全程地图"。
  - 关键词与正文大标题一一对应；末项用中性、不预设结果的词（如"成色几何""他在赌什么"，**不要写"赌局比分"这类预设输赢/过于抽象的词**）。
  - 系列内容上下集各自一条路线，约 5 站为宜，简洁优先。

### 5. 嘉宾背景

- **只讲"人"**：身份 / 头衔（留最有分量的 3 个左右，不堆砌）、凭什么有资格谈本期话题、当前在做什么。
- 与第 4 部分严格分工，零重复。
- 所有履历必须公开可核实，不得编造。
- 推荐格式：`姓名｜头衔1、头衔2、头衔3。一句资历 + 一句当下动作 → 一句"为什么是分量最重的声音"。`

### 6. 核心观点

- **保留项**：默认保留本段。是否精简 / 弱化，由用户按稿件实际决定——若开头价值概括已点明脉络、且正文减负装置充分，本段可适当收紧，但**不擅自删除**。
- **按视频时长分档**：
  - 30–40 分钟视频 → 精简到约 **5 条**。
  - 1 小时左右视频 → **5–8 条**。
- **宁可聚类、不要罗列**（8 条零散 → 合并成链）。5 条之间要构成**逻辑链**（判断 → 旧路为何不行 → 新路跑通 → 真正方案 → 终极结论），而非并列知识点。
- 每条：`### N. 有判断力的小标题` + 3-4 句展开，加粗压关键判断。
- 编辑概括性的强断言（如"死路"）可用于标题 / 导语；核心观点正文里若涉及，注意分寸，必要时回退到更贴原话的表述。

### 7. 重构正文（叙事章节）

- 按视频时间线推进，但**按主题重构**（不逐句照搬字幕）。
- **正文不再自行标注时间戳**：直接对照"原始字幕 + 逐字稿"重构内容即可，章节标题与正文中均不放 `[MM:SS]`。
- 章节标题：用 `#`/`##`，**必须是叙事性 / 场景化**，禁用功能性标题（如"关于 X 的讨论"）。
  - 大章节（H1）下若内容过长，拆 2-3 个叙事性小标题（H2），保证节奏均匀。
  - 每个标题与其下内容必须精确对应，杜绝"挂羊头卖狗肉"。
- **媒体占位符**：在对应段落下方嵌入 `🖼️ [图片占位：…]` / `🎞️ [GIF 占位：…]`，对应原视频的关键截图 / 演示画面（用户据此放截图或 GIF）。一篇通常 8-15 个，不泛滥。
- 关键金句、定义用引用块 `>` 或 `💡 aside` 框承载。
- "下集预告"等过渡内容降级为 H2 或引用块，不与内容大章并列 H1。

#### 长文减负装置（6000 字以上硬核稿必备）

长科普文准确不等于读得下去。以下三件套专治"太长、有门槛、读完不知所云"，总增字数不过两三百，却显著拉高完读率：

1. **路线图句**（开头·1 句）：正文第一个大标题之前加一句"行程预告 + 安全承诺"，告诉读者要被带去哪、会经过什么，并降低畏难情绪。
   - 例：「接下来这趟，我们会从一张"蛋糕幻灯片"出发，看 LeCun 如何一步步推到"世界模型"——**不需要任何技术背景，跟着走就行。**」
2. **一句话路标**（中途·2-3 处）：在关键转折的大标题下、正文首句之前，加一句"承上"的过渡，让读者随时知道身处逻辑链第几环。三处足够，不必每节都加（否则碎）。
   - 例：「联合嵌入看起来已经完美绕开了"模糊诅咒"——但它藏着一个能让整个方法失效的致命漏洞。」
   - 例：「前两层，JEPA 都赢得漂亮，但到了第三层，故事开始变得不一样了。」
3. **"说人话"保底句**（硬核段·按需）：全文门槛最高的技术段（矩阵、公式、架构拆解）末尾，加一句让看不懂细节的人也能拿到结论的话，用 `💡 callout` 跳出。
   - 例：「💡 **看不懂上面这些矩阵也没关系，一句话总结**：就是逼着每个神经元各管各的、别都学成一个样，模型既偷不了懒、又能学到真正有用的特征。」

> 此外，正文导航全局靠第 4 段的「路线条」、局部靠这里的「一句话路标」、收尾靠第 8 段的「收束段」，三者构成完整的"导览系统"。

### 8. 结尾收束段（双锚点之二）

- **职责**：读者跟着正文走完后，替他"合上书"——给一个判断，留一个开放结局。**不是知识点罗列，是下结论。**
- **位置**：正文最后，用一个叙事性大标题（如"写在最后：…"）领起。
- **写法**：
  - 先给"两面"的诚实判断（亮眼的一面 + 清醒的一面），用具体事实，不用空泛赞美。
  - 末尾留一个开放问题 / 悬念，把判断权交还读者（如"几年之内就能看到答案"）。
  - 强断言可以有，但要落在事实上，不预设输赢（呼应路线条末项的中性词）。
- **系列稿分工**：上集收"论点 / 立场" + 抛悬念给下集；下集收"论据 / 现状盘点" + 开放结局。

### 系列上下集处理规范

当一期内容拆成上 / 下集（或多篇系列）时：

- **嘉宾背景跨集完全一致**：同一段、一字不改，强化人物印象（不必每集换写法）。
- **承接句（上抛下接）**：上集结尾抛出的悬念，下集开头第一段必须用一句话"接住"（如"上集结尾我们留了一个问题：…这一集，他给了答案"）。形成"抛—接"闭环。
- **结构线索贯穿**：若正文用"第一层 / 第二层 / 第三层"这类层级线索，必须贯彻到底、不中途脱轨。
- **双向回链**：上集指向下集、下集回指上集；与对照系列（如其他嘉宾同题）互链。
- **路线条各集独立**：每集一条自己的路线条，不共用。

### 去 AI 味（强制自检）

定稿前扫一遍，删掉一眼就是"AI 写的"的词与句式。**目标是保持冷静、有判断力的编辑语感，不是改成大白话。**

- **禁用包装词**（举例，非穷举）：自洽优雅、中期比分、逆袭、难得地、罕见的、令人瞩目、值得一提的是、不难看出、综上所述、换句话说、某种程度上、在这个意义上。
- **禁用空转连接**："首先 / 其次 / 最后"式机械罗列、"不仅…而且…"滥用、排比句轰炸。
- **原则**：用**具体事实**代替形容词（"16 亿打赢 70 亿"胜过"成绩斐然"）；能删的过渡句直接删；引用原话可口语，但旁白叙述要书面克制。

## 对照字幕核校

定稿前必须执行 **CLAUDE.md → Step 2.5 对照字幕核校**（工作流核心，模式 A / B 通用）。
要点：逐板块核对事实 / 数字 / 专名 / 因果；区分必改硬伤与可选润色；对照 `glossary.md` 统一术语；每批给"✅无硬伤 / ⚠️N 处硬伤"总判定。详见 CLAUDE.md。

## 标题生成与文件命名

- 中文标题格式：`嘉宾姓名｜核心主题描述`（全角分隔符 `｜`；含 `/`、`:` 等非法字符替换为 `-` 或空格；不用下划线拼接）。
- 文件名：`<生成的中文标题> - 图文精读.md`
- 存放：`output/<生成的中文标题>/`，与 `transcript.json` 同目录。
- 系列内容（上 / 下集）各自独立目录，但在导读区**双向回链**。

## 输出语言
- 全文简体中文；高共识词保留英文（Token、Agent、JEPA、GPU、API、Transformer…）；
- 公司 / 产品 / 项目代号保留英文原名；新概念首次出现时括注中文解释。
- 对重要 AI 概念，在正文首次提及时用 Obsidian 双链 `[[ ]]` 标记为知识节点（参照 CLAUDE.md Step 3 标签规则）。
