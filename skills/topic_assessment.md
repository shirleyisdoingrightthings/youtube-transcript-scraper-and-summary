# 选题预判规范（Topic Assessment · 编辑视角加权打分）

> **调用时机**：用户**只发来 YouTube 链接、尚未说"开始转录"**时（CLAUDE.md 阶段 -1）。此时不要抓字幕，先以资深编辑身份给视频做一次选题判断，在对话框给出加权分与结论，**然后停住，等用户说"开始转录"**才进入正式工作流。用户若直接说"开始转录"或带模式关键词催促，则跳过本步。
> **目的**：在投入抓取 / 精编之前先判断"这选题值不值得做、怎么做"，把判断前置，减少无效工作。

---

## 一、先做两件描述性判断（必给，不打分）

- **受众**：直接点出这期主要写给谁。（例：技术团队管理层 / engineering leaders，尤其关心"怎么运作一支 AI 化团队"的人）
- **定位**：这期视频是什么、属于哪类。（例：一线管理者的"操作手册"式访谈精读，实操偏管理；或 科普向 / 战略向 / 实操向）

判断依据：视频标题、嘉宾 / 来源、简介；必要时用 WebSearch / WebFetch 快速搜一下嘉宾与议题背景。**此阶段不抓全字幕。**

---

## 二、四维加权打分（每项满分 10，默认等权 25%，综合分保留一位小数）

1. **话题传播度**：嘉宾 / 来源影响力与话题性、议题热度与时效、有没有反差 / 金句 / 数字猛料（如 8x、估值、裁员）、观点是否鲜明、可截图可引用的记忆点。
2. **内容可沉淀性**：有没有可迁移的方法论 / 框架 / 原则（读者能抄走用）、信息密度、半年后还值不值得读、是否填补被忽视的视角。
3. **可扩展性 / 辐射广度**（看五点）：① **圈层穿透**——只对小圈层有用，还是能辐射到管理者 / 产品 / 创业者乃至非技术大众；② **科普潜力**——能否降门槛改写成大众科普且仍有价值；③ **衍生角度**——能拆出几条独立选题 / 传播角度（金句卡、方法论、故事、对照）；④ **跨场景复用**——框架能否迁移到别的行业 / 读者，不绑死在单一语境；⑤ **长尾搜索**——是否覆盖会被反复搜的话题。
4. **独家一手性**：是否主创 / 内部 / 当事人视角，难被复制；越独家越高。

> 权重默认 25/25/25/25，可按用户当期偏好调整（调整时显式报出所用权重）。

---

## 三、可行性标注（不进质量分，只影响"做 / 不做"与排期）

- **制作成本**：字幕可用性（官方逐字稿可用 = 低成本）、时长、需不需要大量事实核查 / 专名核对 → 标 **低 / 中 / 高**。
- **可视化潜力**：有没有幻灯片 / 图表 / demo 可截图（正面项，直接影响成稿质量与工作量）。

---

## 四、输出格式（对话框里给用户）

```
📊 选题预判：<视频标题> | 嘉宾/来源 | 时长·字幕可用性
受众：<写给谁>
定位：<是什么>
① 话题传播度 X/10 — 理由（角度…）
② 内容可沉淀性 Y/10 — 理由
③ 可扩展性/辐射广度 Z/10 — 理由
④ 独家一手性 W/10 — 理由
加权（权重 25/25/25/25）→ 综合 N.N/10
可行性：制作成本 低/中/高；可视化潜力 …
编辑结论：值得做 / 可做 / 可不做；建议模式（图文精读 or 逐字稿）；主打角度与潜在钩子；可破圈的角度。
→ 等你说"开始转录"再进入全流程。
```

给完判断后**停住等待**用户指令，不要擅自进入抓取。
